## Coverage Reporter Skill

Generate, analyze, and report on test coverage metrics for quality assurance.

### When to Use
- After test-runner completes execution
- To verify coverage meets project thresholds
- Before marking an issue as complete
- For PR documentation

### Prerequisites
Use MCP `test-coverage` to generate raw coverage data:
```bash
npm run test:coverage
# or
pytest --cov=src --cov-report=json
```

### Step 1: Collect Coverage Data

#### JavaScript/TypeScript (Istanbul/NYC/Vitest)
Parse `coverage/coverage-summary.json`:
```json
{
  "total": {
    "statements": { "total": 500, "covered": 425, "pct": 85 },
    "branches": { "total": 200, "covered": 156, "pct": 78 },
    "functions": { "total": 100, "covered": 90, "pct": 90 },
    "lines": { "total": 480, "covered": 408, "pct": 85 }
  }
}
```

#### Python (pytest-cov)
Parse `coverage.json` or use:
```bash
coverage json -o coverage.json
```

### Step 2: Analyze Coverage

#### Overall Metrics
- Statement coverage: % of statements executed
- Branch coverage: % of branches (if/else) taken
- Function coverage: % of functions called
- Line coverage: % of lines executed

#### New Code Coverage
Focus on files changed in this issue:
```bash
git diff --name-only main...HEAD | grep -E '\.(ts|js|py|go)$'
```

Calculate coverage only for these files.

### Step 3: Identify Gaps

#### Uncovered Lines
List files with coverage below threshold:
```markdown
### Coverage Gaps

| File | Coverage | Missing Lines |
|------|----------|---------------|
| src/services/payment.ts | 62% | 45-52, 78-85, 102-110 |
| src/utils/validator.ts | 71% | 12-15, 34-38 |
```

#### Critical Uncovered Code
Flag high-risk uncovered areas:
- Error handling paths
- Security-sensitive functions
- Payment/financial logic
- Authentication code

### Step 5: Gate Check

**PASS Criteria:**
- Overall line coverage >= threshold (default 80%)
- New code coverage >= threshold
- No critical security code uncovered

**FAIL Criteria:**
- Coverage below threshold
- Critical paths uncovered
- Security-sensitive code without tests

### Output Format
```

**File:** agents/logs/coverage-42-20240315-143022.md
**Issue:** #42

### Summary
- Overall: 85.2% (threshold: 80%) ✓
- New Code: 86% ✓
- Branch: 78.4% ⚠ (below threshold)

### Verdict: CONDITIONAL PASS
Branch coverage needs improvement but not blocking.

### Action Items:
1. Add tests for tokenService error handling (HIGH)
2. Improve branch coverage in auth module (MEDIUM)
```

### Integration
- Uses: MCP `test-coverage`
- Receives data from: `test-runner`
- Reports saved to: `agents/logs/`
- Referenced by: `create-pr`, `tester`
