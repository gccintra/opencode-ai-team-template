Persist test execution results to structured markdown files for traceability, auditing, and PR documentation.

- After test-runner completes execution
- When documenting test history for an issue
- Before creating a PR (to attach evidence)

### Creating a Log Entry

#### Step 1: Gather Data
Collect from test-runner:
- Test results (pass/fail per test)
- Coverage metrics
- Timing information
- Error details for failures

#### Step 2: Generate Filename
```
test-run-<issue-num>-<YYYYMMDD-HHMMSS>.md
```
Example: `test-run-42-20240315-143022.md`

#### Step 3: Write Log File
Save to `.opencode/work/logs/` directory

#### Step 4: Update Index
Append to `.opencode/work/logs/index.md` (create the file if it doesn't exist):
```markdown
| Date | Issue | Tests | Result | Log |
|------|-------|-------|--------|-----|
| 2024-03-15 | #42 | 48 | PASS | [test-run-42-20240315-143022.md](./test-run-42-20240315-143022.md) |
```

### Log Retention
- Keep all logs for the duration of the issue
- Archive to `.opencode/work/logs/archive/` after PR merge
- Never delete failure logs

### Output Format
```
## Test Log Created

**File:** .opencode/work/logs/test-run-42-20240315-143022.md
**Issue:** #42
**Result:** PASS (46/48 tests)
**Coverage:** 85.2%

Log ready for PR attachment.
```

### Integration
- Receives data from: `test-runner`
- Used by: `create-pr` (for PR documentation)
- Referenced in: `coverage-reporter`
